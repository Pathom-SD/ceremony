# Ceremony — domain glossary

## Ceremony video

วิดีโอที่อัปโหลดในหัวข้อ **Ceremony videos** (`ceremony-videos`) เท่านั้น — ใช้ในห้องสมุดวิดีโอ (แผงลอย) ไม่ปนกับไฟล์ของแผนกอื่น

## Video delivery

การเล่นวิดีโอในแอปส่งผ่าน **`GET /api/files/:fileId`** (เช่น `?preview=1` สำหรับเล่นในหน้า) — ไม่มี route แยกสำหรับสตรีม และไม่เปิด path บนดิสก์ให้เบราว์เซอร์ โดยเซิร์ฟเวอร์รองรับการส่งแบบ **สตรีม** (รวมการขอช่วงไฟล์เพื่อ seek) ทุกครั้งที่ขอไฟล์วิดีโอ ไม่ว่าจะเล่นในหน้า (`preview=1`) หรือดาวน์โหลด — ใช้กับไฟล์ประเภทวิดีโอเท่านั้น ไม่ครอบคลุม PDF รูป หรือเอกสารอื่น

## Video library grid

ในรายการวิดีโอ (โหมด grid) ยังแสดง **เฟรมแรกเป็นภาพย่อ** ผ่าน `<video preload="metadata">` — ไม่เปลี่ยนเป็นพื้นหลังดำอย่างเดียว

## Ceremony video player

เมื่อเลือกวิดีโอจากห้องสมุดวิดีโอ จะเปิด **ตัวเล่นแบบ theater** — โมดัลกลางจอทับพื้นหลังมืด ไม่เล่นในกล่องเล็กของแผงลอย ขณะเล่น **แผงรายการวิดีโอยังเปิดค้างที่ grid** (ไม่สลับเป็นโหมด play ในแผง)

ตัวเล่นใช้ **ควบคุมแบบกำหนดเอง** (ไม่ใช้ UI ของเบราว์เซอร์) รอบแรกรองรับ: เล่น/หยุด · seek และเวลา · ปิด · ชื่อคลิป · ระดับเสียง/ปิดเสียง · เต็มจอ — เปิด player แล้ว **เล่นทันที** (หลังผู้ใช้เลือกคลิป) — แถบควบคุม **ซ่อนขณะเล่น** และ **โผล่เมื่อขยับเมาส์** (โชว์ค้างเมื่อหยุดชั่วคราว) — รองรับ **คีย์ลัดนำเสนอ** (เช่น เล่น/หยุด ปิด seek เต็มจอ ปิดเสียง) — ปิด player ได้ด้วยปุ่มปิด **คลิกพื้นหลังมืดนอกกล่องวิดีโอ** หรือ `Esc`

## Ceremony video size (typical)

ขนาดวิดีโอต่อไฟล์ในงานจริงมักอยู่ช่วง **ประมาณ 500 MB ถึง 2 GB** — การเล่นในแอปต้องไม่โหลดทั้งไฟล์เข้าหน่วยความจำเซิร์ฟเวอร์

## Video response caching

การตอบกลับวิดีโอจาก API ใช้ **`Cache-Control: private, no-store`** เหมือนไฟล์อื่น — ไม่ cache ข้าม session
